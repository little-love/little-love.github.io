C3-3

C3-过渡效果

前言：设置了过渡效果，可以让效果有持续的时间/延迟时间，这样看起来就像是动画的效果了。

  • 固定格式 transition :all 1s 1s
    第一个时间是持续时间，第二个时间是延迟时间
    如果给需要给很多元素设置过渡效果,伪元素不能通过通配符统一设置过渡效果： *,::before,::after{transition:all 1s 1s}

边框宽高

前言：当给盒子添加边框或内边距，会影响到盒子的大小，但是下面这个方法可以轻松解决，不用每次加减。

  • box-sizing:border-box
    优先考虑元素自身的宽度，如果有padding border，会自动往内压缩。
  • box-sizing:content-box

优先考虑内容的尺寸

阴影效果

前言：通过css样式给元素设置阴影效果，可以让元素更立体，表达的更丰富，通常结合过渡效果一起使用，像动画般。

  • 文字阴影 text-shadow
    • 格式：text-shadow:水平偏移 垂直偏移 阴影偏移量 阴影颜色
    • 层：如果需要一层层写阴影，用逗号隔开即可。
    • eg：text-shadow:10px 10 px 10px orange,20px 20px 20px red;
  • 盒子阴影 box-shadow
    • 格式： box-shadow: 阴影位置 水平偏移 垂直偏移 阴影偏移量 阴影大小 阴影颜色
    • eg: box-shadow:inset 10px 10px10px 10px red inset在盒子里面

旋转效果

  • 2D旋转 格式 transform: rotate(角度) 角度单位：deg 度 turn 圈 转
  • 3D旋转 格式 transform:rotate3d(参数)
    其中前3个参数分别表示旋转的方向x,y,z，第4个参数表示旋转的角度，参数不允许省略

C3-2

C3-选择器

  • 属性选择器
    • 格式：E[attr] 有这个类名/属性的 ，比如class
    • 格式：E[attr=“”] 属性为某个值的
    • 格式：E[attr^=””] 属性值以…开头的
    • 格式：E[attr%=””] 属性值以…结尾的
    • 格式：E[attr*=””] 属性值含有…的
      例子
  • 伪类选择器
    格式：::
    • 计算所有兄弟元素:
      • E::first-child 在父元素下的第一个子节点为e的元素 （如果第一个不是e，不起作用）
      • E::last-child 在父元素下最后一个子节点是e的元素（如果最后一个不是e，不起作用）
      • E::nth-child(索引) 在父元素下具体某个索引的元素 索引从1开始
    • 只考虑某一类:
      • E::first-of-type 在e这类种的第一个元素
      • E::last-of-type 在e这类中的最后一个元素
      • E::nth-of-type(索引) 在e这类中的某个索引的元素 索引从1开始

  • 伪元素选择器
    早期浏览器/框架中使用一个冒号，css3规范是用两个冒号。如果兼容老浏览器，写一个冒号；如果不考虑老浏览器，写两个冒号。

    • E::before
      在双标签的e元素里的最前面
      特点：
      1.必须写content:””; 2. 这个是行内元素，通常结合定位一起使用，也让宽高起作用。
      2 . ::before 和 ::after 必须是给双标签 元素设置，给单标签 元素设置不起效果。比如img a就不行。

    • E::after
      在双标签的e元素里最后面

    • E::first-letter
      e元素里的首字母 需要考虑阅读顺序，阿拉伯文是从右向左阅读的。
    • E::first-line
      e元素里首行

    • E::placeholder
      兼容差，需要添加浏览器内核针对的前缀才可以使用。谷歌： -webkit-input-placeholder

    • E::selection
      e元素被选中的样式 ，比如选中时候的背景颜色，字的颜色。

H5-自定义属性

自定义属性
在框架中会经常看到，规范自定义属性，提高阅读性。

  • 设置自定义属性格式： date-type=”属性值” 或 date-type-color=”属性值”
  • 用js获取自定义属性格式：dom.dataset.type 或者 dom.dataset.typeColor

一般很少会自定义属性，原生的属性也足够了，如果需要设置的，再用这个格式即可。

H5-1

H5的新语义标签

解决命名不规范的问题–标签

如果命名没有语义，或者随便命名，即使没有语法错误，但是不方便维护和管理，阅读性也差，为了解决这个问题，h5使用针对的语义标签来解决这个问题。

  • header 页头
  • main 内容
  • footer 页脚
  • section 区域
  • nav 导航
  • aside 内容里除了文章的部分
  • article 文章内容

1.兼容性：ie8以及以下不支持，解决方法1：条件注释 cc:ie6+tab 可以包括任何代码。 解决方法2：下载兼容包，一次搞定。地址：www.github.com 找到html5shiv.js 下载即可，如果打开网页慢，可以用lantern翻墙使用。

2.这类语义标签通常用于整个网页的整体布局，目前如果是小案例，很少用上，也可以用main标签包着使用。

HTMl

Code 1 HTMl

三层分离

遵循W3c规则，html、css、js三层分离，是最合理的代码处理方式。
Html 是从语义的角度，通过使用合适语义的标签布局网页的结构。
Css是从审美的角度，通过布局和属性的使用，美化网页。
Js是从交互的角度，通过语法给浏览器发出指令，提高用户体验。

Html

定义：超级文本标记语言（HyperText Markup Language），在浏览器上运行的一种标记语言。
它包括有图片、文字、音频、视频、超链接等。标记语言就是给文本加上有语义的标签。
